# CS320-Software-Test-Automation
How can I ensure that my code, program, or software is functional and secure?
To ensure my software is functional and secure, I build validation directly into my classes and verify behavior with structured unit testing. Using JUnit and coverage tools forced me to think beyond “does it run” and instead ask whether it handles edge cases, invalid input, and boundary conditions correctly. Security starts with enforcing constraints consistently and preventing bad data from ever entering the system.

How do I interpret user needs and incorporate them into a program?
When interpreting user needs, I slow down and break requirements into clear, testable behaviors before writing code. I map each requirement to a specific class responsibility so nothing feels vague or assumed. If I cannot test it, I revisit the requirement. This keeps my implementation aligned and prevents unnecessary complexity.

How do I approach designing software?
When designing software, I focus on clarity, separation of concerns, and maintainability. I build small, focused classes, implement validation early, and let testing shape my design rather than treating it as an afterthought. My approach is incremental and intentional, prioritizing clean structure and long-term reliability over quick fixes.
